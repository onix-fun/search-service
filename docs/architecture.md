# Архитектура Search Service

## Компоненты
1. **Search API (HTTP):** Высокоскоростной прокси к Meilisearch с проверкой прав.
2. **gRPC API:** Внутренний интерфейс для управления индексами.
3. **Indexer Worker:** Потребляет события из RabbitMQ и обновляет данные в Meilisearch.
4. **Meilisearch:** Основной поисковый движок.
5. **Redis:** Используется для хранения ревизий документов и Leader Election между инстансами индексатора.

## Процесс индексации
1. Изменения в данных (например, новые посты) приходят в RabbitMQ.
2. Индексатор забирает сообщение, проверяет ревизию (чтобы не индексировать старые данные).
3. Индексатор применяет enrichment (транслит и т.д.) и отправляет батч в Meilisearch.
